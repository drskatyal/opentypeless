# OpenTypeless v1.1.49 Multilingual README Refresh Design

## Objective

Refresh the public README set for v1.1.49 without redesigning the repository landing page. Preserve the current information architecture and visual direction while removing stale TalkMore branding and bringing every translated README up to the same release-level product story.

## Scope

The change covers all 18 public README files:

- `README.md`
- `README_zh.md`
- `README_ja.md`
- `README_ko.md`
- `README_es.md`
- `README_fr.md`
- `README_de.md`
- `README_pt.md`
- `README_ru.md`
- `README_ar.md`
- `README_hi.md`
- `README_it.md`
- `README_tr.md`
- `README_vi.md`
- `README_th.md`
- `README_id.md`
- `README_pl.md`
- `README_nl.md`

No website code, release workflow, application code, GitHub metadata, PRs, or external publishing is included in this subproject.

## Content Design

### English README

Keep the existing heading order, positioning, badges, calls to action, v1.1.49 feature copy, Visual Tour, installation instructions, architecture, roadmap, FAQ, and community sections.

Make only these visual corrections:

1. Remove the top `website-hero.png` image because the embedded desktop mockup still presents TalkMore as the product name.
2. Keep `v1.1.49-app-context-showcase.jpg` as the first Visual Tour image, as previously approved.
3. Remove the legacy `voice-flow-demo.gif` from the Visual Tour because it contains the old TalkMore product UI.
4. Keep the existing v1.1.49 app-aware polish and dictionary screenshots.

### Translated READMEs

Preserve each file's existing language navigation, translated voice, installation guidance, configuration content, architecture, roadmap, FAQ, community links, and license section.

For each translated README:

1. Replace the legacy TalkMore demo image with `v1.1.49-app-context-showcase.jpg`.
2. Add a localized `What's New in v1.1.49` section near the top.
3. Cover the same six release themes as the English README:
   - local app-aware writing;
   - multilingual voice intent routing;
   - multiple shortcuts per workflow;
   - switchable translation targets;
   - dictionary import/export and correction rules;
   - per-app style mappings.
4. Include the same privacy boundary: app detection and mappings are local; raw window titles and document contents are not sent as app context or stored in history.
5. Add the approved app-aware polish and dictionary screenshots without creating new visual assets.
6. Add app-aware writing and multiple shortcuts to comparison or feature content when those rows are absent.

Translations should be natural product copy in the target language rather than word-for-word English substitutions. Product names, provider names, shortcut labels, filenames, and code identifiers remain unchanged.

## Brand Rules

- `OpenTypeless` is the only user-facing desktop product name.
- `TalkMore` must not appear as the desktop product name in README text, image references, alt text, or captions.
- Existing historical filenames may remain in the repository, but no refreshed README may render a stale TalkMore asset.
- Competitor names such as Typeless, Wispr Flow, and Superwhisper remain where they are used for factual positioning.

## Non-Goals

- No README redesign or new badge system.
- No new screenshots, generated artwork, download metrics, or social-preview design.
- No changes to GitHub Actions billing, Windows signing, PR triage, or contribution policy.
- No website analytics or outbound posting in this subproject.

## Review Handoff

After implementation, open the rendered English README first and provide direct links to the changed translated README files. Do not push to `main` until the user approves the result.
