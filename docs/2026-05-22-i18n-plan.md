# Full UI i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace all ~136 hardcoded user-facing strings with i18next `t()` calls, add 8 new languages (ja, ko, fr, de, es, pt, ru, it), and make the Rust tray menu respond to the selected UI language.

**Architecture:** Extend existing i18next setup. Frontend migrates hardcoded strings to `t()` keys. Rust tray menu reads `ui_language` from Tauri store and looks up labels from a built-in HashMap. Frontend triggers tray refresh via a new `refresh_tray_labels` command after language switch.

**Tech Stack:** i18next, react-i18next, Tauri store, Rust HashMap
